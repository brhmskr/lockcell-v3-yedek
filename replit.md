# MES Uretim Takip Sistemi

## Overview
This project is an operation-based Manufacturing Execution System (MES) with integrated ERP capabilities. Its primary purpose is to provide real-time production tracking, comprehensive cost management, and detailed profitability analysis for manufacturing operations. The system aims to enhance efficiency, reduce waste, and improve decision-making through accurate data and insights.

## System Architecture
The system is built with a modern web stack:
- **Frontend**: React, utilizing TanStack Query for server state management, `shadcn/ui` for components, and Tailwind CSS for styling, including dark mode support.
- **Backend**: Express.js, providing a RESTful API.
- **Database**: PostgreSQL, managed with Drizzle ORM.
- **Authentication**: Session-based authentication using `express-session` and `connect-pg-simple`. It supports both admin (username/password) and operator (machine selection, operator name, 4-digit PIN) logins. Sessions are persistent with a 24-hour cookie.
- **UI/UX**: Features a tabbed admin dashboard, real-time machine status cards with images, a dual-tab login page, and a dedicated public TV dashboard for factory-wide visualization. The design incorporates `shadcn/ui` components and Tailwind CSS for a consistent and responsive experience, with theme-aware logos for dark and light modes.
- **Key Features**:
    - **Operator Panel**: Manages production workflow (start, stop, resume, finish), stop reason tracking, quantity entry with quantity guard validation (prevents exceeding target quantity), machine-centric shift handover (vardiya devralma - operator can take over active jobs on a machine from another operator), machine-scoped chat with file sharing, PDF document viewing, and technical drawing viewing with revision acknowledgment (must acknowledge latest revision before BASLAT).
    - **Admin Dashboard**: Offers 12 distinct sections for comprehensive management:
        - **Overview**: Real-time machine status, work order tracking, production statistics, and an executive summary with key financial metrics (e.g., efficiency loss cost, profitable orders, tool cost alarms) all normalized to EUR.
        - **Tezgah Chat**: Real-time messaging between admins and machines with unread counts, file/image sharing, and notifications.
        - **Management Sections**: CRUD operations for Machines (with hourly cost, image upload), Operations, Personnel, Operator Assignments, Work Orders (including bulk Excel upload with template download, technical drawing upload with revision tracking per work order), Stop Reasons, and Recurring Expenses.
    - **Technical Drawing & Revision Tracking (DMS)**: Upload PDF/PNG/JPG technical drawings per work order. Auto-increments revision numbers, supports revision notes, tracks which operators have acknowledged each revision. Schema: `technical_drawings` (id, workOrderId, fileName, fileUrl, revisionNumber, revisionDate, revisionNote, uploadedBy, isCurrent) + `drawing_acknowledgments` (id, drawingId, userId, acknowledgedAt). API: GET/POST `/api/technical-drawings/:workOrderId`, GET `/:workOrderId/current`, GET `/acknowledgments/:drawingId`, POST `/:drawingId/acknowledge`. Max upload 20MB.
        - **Reporting & Analysis**: Detailed Production Reports (with CSV export), Machine Efficiency (OEE) dashboard with charts, and Work Order Profitability analysis.
    - **Fabrika TV Ekranı (`/tv-dashboard`)**: A public, full-screen dashboard displaying real-time machine statuses, production progress, and work order details using circular SVG progress rings and status-coded gradient cards.
- **Cost & Profitability Model**: All costs are ultimately tracked in EUR. Expenses can be entered in TRY and are automatically converted using live exchange rates. The model accounts for machine hourly costs, allocated overhead from monthly expenses, material costs, and tool costs. Profitability is calculated by comparing `targetPrice` against `Total Cost` (Direct Machine Cost + Overhead Share). Efficiency loss is factored into the profitability calculation.

## External Dependencies
- **PostgreSQL**: Primary database for all application data.
- **TanStack Query**: Used on the frontend for managing server state.
- **Tailwind CSS**: Utility-first CSS framework for styling.
- **Recharts**: Charting library used for data visualization in the admin dashboard.
- **exchangerate-api.com**: Primary API for fetching live EUR/TRY exchange rates.
- **frankfurter.app**: Fallback API for EUR/TRY exchange rates.
- **Replit App Storage (Object Storage)**: Used for file and image uploads (e.g., machine images, chat attachments).
- **express-session**: Middleware for session management.
- **connect-pg-simple**: PostgreSQL-backed session store.
- **Drizzle ORM**: ORM for interacting with PostgreSQL.
- **Node.js crypto**: Used for password hashing.
- **Zod**: Schema declaration and validation library.
